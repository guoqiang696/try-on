## 二、项目背景与痛点

### 2.1 用户侧痛点

- **网购服装决策难**：服装类电商退货率长期高达 50%~60%，核心原因是"不知道穿上身什么样"，图片与实物、模特与本人之间存在巨大的想象鸿沟
- **试错成本高**：线下试衣排队耗时、款式有限；婚礼礼服、旅拍服装、职场着装等低频高价场景，用户缺乏低成本的"预演"手段
- **决策依赖社交**："帮我看看哪件好看"是高频社交行为，但目前只能靠转发商品链接，缺乏直观的上身效果

### 2.2 商家侧痛点

- 服装商品图依赖模特棚拍，单款拍摄成本数百至数千元，上新周期长
- 直播带货中主播实穿换装效率低，一场直播可展示 SKU 有限

### 2.3 运营商转型机遇

公司正推动**从流量运营向 Token 运营转型**：过去经营管道中的字节，未来经营算力中的智能调用。当前缺乏**高频、可计量、面向大众**的 token 消耗场景作为转型样板。AIGC 换装具备"单次调用 token 消耗确定、使用高频、自带社交传播"三大特征，是 token 化运营的理想试验田。

## 三、产品方案

### 3.0 产品当前形态

产品已构建完整的**web端体验系统**，覆盖首页、三大试衣工作台、作品画廊与个人中心全链路，直接可用于用户测试与评审演示。

### 3.1 核心能力

输入人物照片 + 目标衣物图片，通过 AI 生成高保真换装效果。产品定位为**"专属 AI 试衣师"**：用户把衣服交给 AI，AI 认真逐件试好后通知用户查看——以"画质优先"而非"速度优先"建立价值感。

- **三种试衣模式**：预设模特换装（选择平台模特 + 上传衣物）、真人照片试穿（上传生活照 + 目标衣物）、自由风格生成（上传参考图 + 配置人物属性 + 场景风格选择 + 文字描述）
- **积分制消费闭环**：每种试衣模式有明确积分单价（模特试衣 5 积分/次、真人试衣 20 积分/次、自由风格 40 积分/次），提交前展示任务摘要与费用预估，确认后扣减积分并进入后台渲染队列
- **实时进度追踪**：提交后跳转至"我的作品"画廊查看 SVG 环形进度条（0~100%），完成后显示通知红点，生成结果可放大 lightbox 预览、下载、收藏或"再次使用"一键复用
- **三档会员订阅**：Pro（¥29/月，500 积分）、Plus（¥79/月，1,500 积分，推荐档）、Ultra（¥199/月，5,000 积分），积分按月发放未用完可累计，Plus 以上享优先队列与高清生成
- **云网算力承载**：推理部署于电信自有算力（息壤），预生成与批量任务**错峰调度至闲时算力**执行，体现运营商算力精细化运营能力与"云网+算力+AI"一体化底座

### 3.1.1 三种试衣模式（当前已实现）

1. **预设模特换装**：平台预置多位不同风格模特（含身高、风格标签如"休闲""时髦"等），用户选择模特后上传目标衣物图片即可生成。适合快速预览商品上身效果。
2. **真人照片试穿**：用户上传个人生活照 + 目标衣物图片，生成真实的上身试穿效果。适合个人消费决策场景。
3. **自由风格生成**：上传参考图片（可选），配置人物属性（性别、身高、年龄、体型标签如 标准/偏瘦/运动/丰满/壮硕），选择场景风格（如 都市/影棚/户外/晚装 等），填写文字描述（含"自动总结"辅助按钮），由 AI 综合创作。适合创意搭配与风格探索。

三种模式共享右侧**任务摘要面板**，实时展示输入状态、生成数量调节（数量步进器）与预估积分消耗，缺少必填项时「生成」按钮自动置灰引导补全。

### 3.1.2 规划中的进阶交互形态

1. **试衣篮（批量模式）**：用户像逛街一样将多件衣物加入试衣篮，一次提交、一批生成，1~2 分钟后收到整组对比结果——批量摊薄等待感，且天然服务于"多件对比、帮我选"的决策场景
2. **异步任务通知**：单件试穿提交后可关闭页面，生成完成推送通知；查看结果采用"开箱式"揭晓动画，把等待后的揭晓做成情绪高点
3. **今日为你试穿（预生成模式）**：用户录入形象照后，系统在闲时算力上对热门款、收藏款、节令穿搭提前批量生成，用户打开即看到已生成的试穿 feed 流——前台体验"零等待"

### 3.2.1 当前已实现的完整用户流程

1. **首页**（Hero 入口 → 三种试衣方式 Bento 卡片 → 生成案例展示墙）
2. **选择试衣模式**进入对应工作台（三栏布局：左侧参数配置 → 中间素材上传 → 右侧任务摘要）
3. **填写信息并点击生成** → 弹出确认弹窗（展示任务摘要与积分消耗预估，积分不足时禁用确认按钮）
4. **确认后扣减积分** → 自动跳转画廊页 → 实时进度环动画（demo 阶段 8 秒模拟，真实接入后可展示实际推理进度）
5. **生成完成** → 侧栏「我的作品」显示红点通知，结果卡片支持放大 lightbox 查看、下载、收藏、一键复用
6. **个人中心**查看积分余额、生成统计、订阅方案、偏好设置、最近试穿记录

### 3.3 技术方案

- **前端架构**：桌面端带侧栏导航 + 顶部工具栏，移动端适配底部 Tab 栏；毛玻璃设计语言 + Material Symbols 图标系统 + SVG 进度环 + Lightbox 灯箱 + 入场动效
- **前端工程化**：组件化 JS 模块（路由 opc-router.js、gen-flow opc-generation-flow.js、订阅弹窗 opc-subscription-modal.js、lightbox opc-lightbox.js、特效 opc-bits-effects.js），CSS 以设计系统层划分（pages、enhancements、bits-effects、apple-glass），支持亮色/暗色双主题
- **AI 引擎**：基于扩散模型的虚拟换装（Virtual Try-On）技术：人体解析 + 姿态估计 + 衣物特征注入 + 高保真生成
- **后端架构（规划中）**：**异步任务队列 + 消息通知**为核心链路；预生成与批量任务错峰调度至闲时算力，显著摊薄推理成本；高峰期排队削峰并给出预估完成时间
- **持续优化**：模型蒸馏/少步采样、TensorRT 推理优化，生成时长随版本迭代逐步压缩（当前约 1 分钟/张，目标半年内进入 10 秒级）
- **安全合规**：用户照片授权协议、人像数据省内算力域内处理不出域、生成图片嵌入隐式水印与 AIGC 标识，符合《生成式人工智能服务管理暂行办法》要求

## 四、创新点与亮点

1. **场景创新**：将 AIGC 换装从"娱乐玩具"重构为"消费决策工具"，切入电商退货率这一真实产业痛点，惠民价值可量化
2. **体验创新——积分透明、全程可感**：每个试衣操作前有明确的积分预估与任务摘要，提交后实时进度追踪，画廊内置放大、下载、收藏、复用等完整操作闭环，把 AI 生成的"黑盒感"变为可控可感的使用体验
3. **模式创新——Token 化运营样板**：
  - C 端：三档订阅制（Pro ¥29/500 积分 · Plus ¥79/1,500 积分 · Ultra ¥199/5,000 积分），积分未用完可累计——完整复用运营商最成熟的"套餐运营"方法论
  - B 端（规划中）：API 按 token 计费，单价透明、用量可预测，形成第二增长曲线
  - 单次生成算力消耗大、任务可离线，恰好构成**大颗粒、可调度**的 token 消耗模型，客单价逻辑扎实
4. **底座差异化——算力错峰运营**：互联网厂商做应用，运营商做"应用+算力+计费"一体化。预生成任务调度至闲时算力执行，推理成本显著下降——这正是运营商算力精细化经营能力的活样板，构成成本侧竞争壁垒
5. **传播自驱动**：换装结果天然适合社交分享，"生成—分享—回流"形成用户增长与 token 消耗的双轮飞轮
6. **前端工程亮点**：零框架依赖的纯静态 SPA，Hash 路由 + 三栏栅格工作台 + 毛玻璃 UI 体系，模块化 JS 架构清晰可维护，全链路可交互原型可直接用于用户测试与评审演示

## 附录：GPU 算力部署成本与盈利测算

### A.1 GPU 硬件选型对比

虚拟换装推理对显存要求较高（需同时加载人体解析、姿态估计、扩散模型等多个子模型），推荐以下方案：


| 型号           | 显存   | FP16 算力     | 单卡价格（含税）        | 单卡并发  | 适用场景          |
| ------------ | ---- | ----------- | --------------- | ----- | ------------- |
| RTX 4090     | 24GB | 82.6 TFLOPS | ¥13,000~15,000  | 1~2 路 | 小规模验证 / 闲时预生成 |
| RTX 6000 Ada | 48GB | 91.1 TFLOPS | ¥45,000~55,000  | 2~4 路 | 中规模生产         |
| A10          | 24GB | 31.2 TFLOPS | ¥25,000~35,000  | 2~3 路 | 性价比之选         |
| L40S         | 48GB | 91.6 TFLOPS | ¥40,000~60,000  | 3~5 路 | 推荐主力          |
| A100 80GB    | 80GB | 312 TFLOPS  | ¥80,000~100,000 | 5~8 路 | 高并发线上         |


**推荐方案**：初期 2 卡 L40S 起步（约 ¥10 万），兼顾显存与并发，后续按量扩展。预生成任务可搭配更便宜的 RTX 4090 做闲时算力池。

### A.2 推理成本核算

**基础参数**（以 L40S 为例，扩散模型换装推理）：

- 单次生成耗时：60 秒/张（未优化）→ 15 秒/张（TensorRT + 少步采样优化后）
- 单卡理论日产能：24h × 3600s ÷ 15s = 5,760 张
- 考虑排队/故障/冷启动，有效利用率按 70% 计：约 **4,000 张/卡/天**
- 单卡年产能：约 **146 万张**

**电费**（单卡满载约 350W，含服务器其他功耗约 500W）：

- 日电费：0.5kW × 24h × ¥0.8/kWh（工业电价）≈ ¥9.6
- 年电费：约 ¥3,500/卡

**均摊成本**（按 3 年折旧）：


| 成本项                    | 年费用（单卡）     | 摊至每张图          |
| ---------------------- | ----------- | -------------- |
| 硬件折旧（L40S ¥50,000 ÷ 3） | ¥16,667     | ¥0.011         |
| 电费                     | ¥3,500      | ¥0.002         |
| 机房/带宽/运维               | ¥5,000      | ¥0.003         |
| **合计**                 | **¥25,167** | **≈ ¥0.017/张** |


即：**单张换装图的边际算力成本约 1.7 分钱**。这是自有算力、优化到位后的理想值。初期未优化阶段（60 秒/张）约 7 分钱/张。

### A.5 关键风险与降本策略

1. **初期推理速度慢**：未优化模型 60 秒/张，需排队导致体验差 → 用闲时预生成 feed（"今日为你试穿"）遮盖延迟，前台零等待
2. **高并发峰值**：晚 8~10 点高峰，GPU 不够 → 闲时算力池（RTX 4090）+ 排队削峰 + 预生成热门款缓冲
3. **模型持续迭代成本**：每次模型升级需 GPU 重训练 → 利用息壤平台按需租用训练算力，不囤硬件
4. **用户活跃度低于预期**：GPU 闲置 → 闲时自动对热门款做预生成，填满算力缺口，产能不浪费
