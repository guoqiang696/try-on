# AI 智能试衣门户 · 功能介绍文档

> 本文档面向产品、设计、研发与商业团队，描述「智能试衣」的定位、三大核心体验、安全围栏与商业化设计。

---

## 1. 产品定位：「能力」到「门户」

智能试衣是一个**面向 C 端的完整 AI 试衣门户**——用户进入即拥有一个「私人 AI 试衣间」，可在三种体验模式之间自由切换，全程以图片为中心，不暴露任何 API / 接口 / 模型概念。

门户由四层能力构成：


| 层       | 内容                      | 解决的问题           |
| ------- | ----------------------- | --------------- |
| **体验层** | 模特试衣 / 真人试衣 / 自由风格 三种模式 | 「我想看衣服怎么穿」的不同诉求 |
| **过程层** | 上传/解析/阶段式进度/成功/失败的全状态反馈 | 「生成要等多久、为什么失败」  |
| **资产层** | 我的作品、收藏、历史、原图对比         | 「我生成的东西去哪了」     |
| **信任层** | 安全围栏 + 商业化（积分计费）        | 「我的脸安全吗、要花多少钱」  |


> 设计主线：**明亮科技风 + 轻时尚感**，玻璃拟态（glassmorphism）作为统一视觉语言，蓝青渐变为品牌主色。

## 2. 三大核心体验模式

三种模式共享同一套**页面骨架**：左侧主输入 → 中间次输入 → 右侧任务摘要（详见第 5 节统一规范）。区别只在输入内容，不在布局结构。

### 2.1 模特试衣（Model Try-On）

- **一句话**：不想露脸，只想快速看某件衣服穿在专业模特身上的效果。
- **核心输入**：选预设模特 → 上传衣物图 / 粘贴商品链接 → 选衣物类型 → 设张数。
- **适合**：电商卖家、想种草的逛街党、不愿上传本人照片的用户。
- **算力系数**：1.0（基准）。

### 2.2 真人试衣（Person Try-On）

- **一句话**：上传自己的生活照，看「我」穿上这件衣服的真实效果。
- **核心输入**：上传本人照片（+身高/体重/体型）→ 上传目标衣物 → 选类型 → 设张数。
- **适合**：购买前预览、社交分享、对版型敏感的用户。
- **算力系数**：1.2（需人像分析，成本更高）。
- **⚠️ 高敏感模式**：涉及人脸生活照，**必须**走完整安全围栏（见第 3 节），授权勾选为提交前置条件。

### 2.3 自由风格（Free Style）

- **一句话**：基于一张图或一段描述，生成不同场景、风格下的穿搭大片。
- **核心输入**：参考图（可无图）→ 生成目标 → 场景 → 风格 → 自由描述 → 设张数。
- **适合**：内容创作者、找穿搭灵感、做轻量商品展示图。
- **算力系数**：1.0。
- **预期管理**：生成前明确提示「结果为 AI 创意生成，不等同真实试穿」。

### 模式对比一览


| 维度      | 模特试衣    | 真人试衣        | 自由风格          |
| ------- | ------- | ----------- | ------------- |
| 是否上传本人照 | 否       | **是（强制授权）** | 可选            |
| 主输入     | 模特 + 衣物 | 本人照 + 衣物    | 参考图/描述 + 场景风格 |
| 结果性质    | 换装预览    | 真实试穿        | 创意大片          |
| 算力系数    | 1.0     | 1.2         | 1.0           |
| 安全等级    | 中       | **高**       | 中             |

## 3. 安全围栏设计（管控敏感信息生成）

安全围栏是门户的「信任层」，目标是：**在不打断主流程的前提下，前置拦截敏感内容、最小化敏感数据留存、对生成结果做可追溯标识**。围栏贯穿「上传 → 生成 → 结果 → 留存」全链路。

### 3.1 四道围栏

```
①入口授权  →  ②内容审核  →  ③生成约束  →  ④结果标识
（上传前）     （上传/提交时）   （生成中）      （生成后）
```

#### ① 入口授权（上传前）

- 真人试衣上传本人照片前，**强制勾选**：「我确认照片为本人，或已获得照片中人物的肖像授权」。未勾选则提交按钮禁用。
- 授权文案明确用途边界：**照片仅用于本次试穿生成，不用于训练、不对外展示**。
- 链接输入框旁标注「支持的链接来源」，避免对解析能力的过高预期。

#### ② 内容审核（上传/提交时）

对图片与文字描述做敏感内容识别，命中即拦截，并给出**可执行的下一步**（不暴露具体审核规则）：


| 拦截类型           | 处理          |
| -------------- | ----------- |
| 疑似未成年人肖像       | 拒绝生成，提示更换照片 |
| 公众人物 / 他人肖像滥用  | 结合授权声明约束，拒绝 |
| 涉黄 / 涉暴 / 违法违规 | 拒绝生成，提示修改   |
| 敏感文字描述（自由风格）   | 高亮提示，引导修改描述 |


#### ③ 生成约束（生成中）

- 模型层面限制生成方向：禁止生成暴露、违规、误导性内容。
- 真人模式仅做「换装」，不改变人脸身份特征，避免「换脸」滥用。

#### ④ 结果标识（生成后）

- 所有 AI 生成图统一加**可见「AI 生成」角标**。
- 下载 / 分享的图片附加水印（可见或隐形），满足合规与可追溯要求。

### 3.2 数据生命周期（最小留存）


| 数据        | 留存策略                        |
| --------- | --------------------------- |
| 原图（人脸/衣物） | 仅用于本次生成，生成完成后短期保留供重试，超期自动清除 |
| 结果图       | 进入「我的作品」，用户可随时删除            |
| 删除作品      | 即时生效，同步删除关联原图与结果图           |
| 账号注销      | 清除该用户全部原图、结果图与任务记录          |


个人中心提供「隐私设置」入口，可查看 / 管理已上传内容、一键清除历史。

### 3.3 UI 落点

- **真人试衣页**：授权勾选条（提交前置）、底部隐私承诺、上传质量指南。
- **自由风格页**：描述框下方「AI 创意生成」声明、敏感词提示。
- **结果页 / 作品库 / 状态页**：每张图左下角「AI 生成」角标。
- **状态页失败态**：「内容安全拦截」错误卡，含原因 + 重新上传 / 修改描述按钮。

## 4. 消费与商业化思考

门户使用**积分体系**。

### 4.1 计价模型

```
本次消耗 = 单张基础价 × 生成张数 × 模式系数
```

- 模式系数体现算力成本差异：模特 1.0 / 真人 1.2 / 自由 1.0。
- 生成配置面板**实时展示预计消耗**，随张数即时更新。
- 示例：单张基础价 5 积分，真人试衣生成 4 张 = 5 × 4 × 1.2 = 24 积分。

### 4.2 扣费时机（用户公平优先）


| 时机   | 规则                 |
| ---- | ------------------ |
| 提交前  | 展示预计消耗 + 校验余额是否充足  |
| 成功后  | 按**实际成功张数**实扣      |
| 失败时  | 不扣费                |
| 部分成功 | 仅对成功张数计费，失败部分退还/不扣 |
| 用户取消 | 已完成张数照常计费，未完成不扣    |


### 4.3 商业化阶梯


| 阶段            | 形态                             | 目标                 |
| ------------- | ------------------------------ | ------------------ |
| **MVP（最小闭环）** | 新用户免费额度 + 充值占位入口（可暂不接入真实支付）    | 让「余额为 0」不是死路，闭环可演示 |
| **成长期**       | 真实支付、积分套餐、会员等级（VIP 折扣/高清/优先队列） | 提升 ARPU            |
| **成熟期**       | 创作者分成、商家批量套餐、品牌联名场景模板          | 拓展 B 端与生态          |


### 4.4 余额为 0 的闭环

失败态「积分不足」必须提供出路：展示「需要 X / 当前 Y」→「获取额度 / 充值」按钮 → 跳转个人中心积分区。这是 MVP 的硬性要求，避免流程死路。

### 4.5 变现增长点（思考方向）

- **会员订阅**：高清生成、去水印（合规前提下）、优先队列、更多模特库。
- **场景模板付费包**：婚礼 / 旅拍 / 职场等精修模板。
- **B 端电商**：商家上传 SKU 批量换装出图，按量计费。
- **裂变**：分享海报带品牌水印 → 拉新返积分。
